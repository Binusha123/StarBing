# StarBing Travel Logistics  Dashboard

A **Power BI-based logistics analytics dashboard** designed to monitor and analyze delivery operations across hubs, drivers, and vehicles. The dashboard transforms raw logistics data into **interactive visual insights** that help organizations improve operational efficiency and decision making.

---

# Project Overview

Modern logistics companies handle large volumes of deliveries involving multiple hubs, drivers, and vehicle fleets. Managing these operations without proper analytics makes it difficult to track performance, identify delays, and optimize resources.

This project develops a **Power BI dashboard** that integrates logistics data to provide a **centralized view of operational performance**.

The dashboard enables managers to monitor:

* Delivery performance
* Hub capacity utilization
* Driver efficiency
* Fleet availability
* Operational bottlenecks

---

# Key Features

* Interactive Power BI dashboard
* Real-time logistics performance monitoring
* Hub capacity vs order analysis
* Driver performance and delay analysis
* Fleet utilization insights
* KPI tracking for operational efficiency

---

# Dashboard KPIs

The dashboard tracks the following key performance indicators:

* **Total Orders:** 1172 deliveries processed
* **On-Time Delivery Rate:** 79.10%
* **Average Delivery Time:** 35.96 hours

These KPIs provide a quick overview of logistics efficiency and delivery performance.

---

# Dashboard Visualizations

The dashboard includes several visual analytics components:

### KPI Cards

Displays key operational metrics including total orders, on-time delivery rate, and average delivery time.

### Hub Capacity vs Orders

A clustered bar chart comparing hub capacity with actual orders processed to identify overloaded or underutilized hubs.

### Hub Performance Ranking

Ranks hubs based on on-time delivery rate to identify top-performing and underperforming hubs.

### Driver Experience vs Performance

Scatter plot analyzing the relationship between driver experience and performance ratings.

### Delay Rate by Driver

Bar chart highlighting drivers responsible for the highest delivery delays.

### Vehicle Fleet Analysis

Donut chart displaying the distribution of active vs maintenance vehicles.

### Orders by Vehicle Model

Bar chart comparing delivery volumes handled by different vehicle models.

# Key Insights

From the dashboard analysis:

* Some hubs process significantly more orders than others, indicating uneven workload distribution.
* Drivers with higher experience tend to demonstrate better performance ratings.
* A small number of drivers contribute to higher delivery delay rates.
* Most vehicles are active while a smaller portion remains under maintenance, impacting fleet availability.
